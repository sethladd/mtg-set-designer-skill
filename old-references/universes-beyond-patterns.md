# Universes Beyond: IP Adaptation Patterns

This reference distills lessons from how Wizards of the Coast adapted external IPs into Magic cards across Universes Beyond products: Lord of the Rings: Tales of Middle-earth (2023), Warhammer 40,000 (2022), Doctor Who (2023), Fallout (2024), and Final Fantasy (2025). These patterns should guide Phase 0.5 (Theme Research), Phase 2 (Worldbuilding), Phase 3 (Mechanics), and Phase 5 (Card File) whenever the set adapts an existing IP.

## Lesson 1: Card naming follows a consistent formula

Across all UB sets, WotC uses a predictable naming pattern for different card categories:

**Named characters: "Name, Title/Descriptor"**
- Aragorn, King of Gondor
- Marneus Calgar (no title needed — the name is iconic enough)
- Caesar, Legion's Emperor
- Cloud, Midgar Mercenary
- Piper Wright, Publick Reporter
- Szarekh, the Silent King

The descriptor captures the character's *role or state at a specific story moment*, not a generic adjective. "Gandalf the Grey" and "Gandalf, White Rider" are different cards because they're different story moments. This is how multiple versions of the same character coexist.

**Rule: Major characters get 2–3 versions across the set, each at a different point in their arc.** Aragorn has Company Leader, King of Gondor, and other versions. Cloud has Midgar Mercenary, Ex-SOLDIER, and Planet's Champion. Each version should feel mechanically distinct — different colors, abilities, or strategies.

**Non-character cards: Direct IP terminology**
- Spells named after iconic quotes or events: "You Cannot Pass!", "GROND! GROND! GROND!", "Nuclear Fallout"
- Locations as card names: Mount Doom, Gold Saucer, The Dead Marshes
- Game systems as card names: V.A.T.S., Overencumbered, Pip-Boy
- Objects/items: Glamdring, Wasteland Survival Guide

**Rule: Never invent a generic Magic name when an IP-specific name exists.** A removal spell in a LotR set is "Isildur's Fateful Strike," not "Heroic Smite." A vehicle in a Fallout set is "Brotherhood Vertibird," not "Armored Transport." Every card name is an opportunity to make the IP present.

**Common/uncommon creature naming:** Even generic creatures use IP terminology for their type lines and names. A soldier in a Warhammer set is "Ultramarines Honour Guard," not "Elite Soldier." A bird in a Final Fantasy set is "Chocobo," not "Golden Steed."

## Lesson 2: Flavor text strategy varies by IP type

WotC uses three distinct flavor text approaches depending on the source material:

**Direct quotes from the source (literary IPs):** Tales of Middle-earth used *exclusively* Tolkien's own words for flavor text — no original text at all. This worked because Tolkien's prose is exceptional and the fan base reveres the text. Community response was overwhelmingly positive.

**Adapted quotes and original text (mixed IPs):** Warhammer 40K searched through "dozens of Warhammer books for suitable quotes" and mixed them with original text. Iconic phrases like "In the grim darkness of the far future, there is only war" appeared directly. Where no suitable quote existed, original text in the IP's voice was created.

**Minimal quotes, mechanic-driven flavor (game IPs):** Fallout and Final Fantasy used less direct quotation (games have less quotable prose than books) and more original text that describes the game world. Game-specific terminology fills the flavor role instead ("Gary!" from Fallout, "Exterminate!" from Doctor Who).

**Rule for set design:** During Phase 0.5, identify which flavor text strategy fits the IP:

- If the IP has a strong literary tradition with beloved prose → use direct quotes heavily
- If the IP has iconic catchphrases but limited prose → mix quotes with original text in the IP's voice
- If the IP is primarily visual/interactive (games, animation) → write original flavor text using the IP's terminology, tone, and worldview
- In all cases, flavor text should sound like it *comes from* the IP's world, never from generic fantasy

## Lesson 3: Character density is higher than you'd expect

The most successful UB sets have *far* more named characters than typical Magic sets:

| Set | Legendary Creatures | Total Cards | Character Density |
|-----|-------------------|-------------|-------------------|
| Tales of Middle-earth | 75 | ~281 | ~27% |
| Doctor Who | ~80+ | ~200 | ~40% |
| Final Fantasy | 60+ | ~261 | ~23% |
| Fallout | 52 | ~164 | ~32% |

**Rule: For character-rich IP sets, target 20–30% of the set as named characters.** This is dramatically higher than a typical Magic set (~5–10% named legendaries). The reasoning is simple: fans come to a UB set *to see their favorite characters*. Every generic creature is a missed opportunity.

**Character version strategy (from LotR):**
- Tier 1 characters (protagonists/main villains): 2–3 versions at rare/mythic, showing different story moments
- Tier 2 characters (major supporting cast): 1 version at rare
- Tier 3 characters (minor but recognized): 1 version at uncommon
- Tier 4 characters (deep cuts for fans): 1 version at uncommon or referenced in flavor text/names

**The Nazgûl precedent (from LotR):** Nine unique art treatments for a single card name, with a special rule allowing up to nine copies. When the IP has a group identity (the Nine, the Mane Six, the Thirteen Doctors), consider whether a single card with a special copy rule or a cycle of individual cards serves it better.

**Fallout's "no player avatar" rule:** For IPs where the protagonist is a player-insert character (video game RPGs), the set used NPCs as legendaries rather than the player character. The Lone Wanderer, The Chosen One, and The Courier were deliberately excluded because they represent *the player*, not a character. This preserves player agency and avoids the "that's not MY character" problem.

## Lesson 4: Mechanics should translate the IP's systems, not just its flavor

The strongest UB mechanics are ones that *mechanically recreate* something the IP does, not just reference it thematically:

**The Ring Tempts You (LotR):** A progressive emblem that gains abilities over time, representing the One Ring's growing corruption. The Ring-bearer gets stronger but becomes a target — exactly like in the books. This isn't "a mechanic that references the Ring"; it IS the Ring, expressed as game rules.

**Rad Counters (Fallout):** Players accumulate radiation that triggers at pre-combat: mill cards equal to rad counters, lose life for each nonland milled. This mechanically recreates radiation damage as a slow, persistent threat that degrades your resources — exactly how radiation works in Fallout games.

**Saga Creatures (Final Fantasy):** Summons in Final Fantasy are magical beings you call on in battle. The set created a new card type — enchantment creatures with Saga chapters — that mechanically recreates the summon sequence: call the summon, it acts over multiple turns, then sticks around as a creature.

**Doctor's Companion (Doctor Who):** Each Doctor pairs with specific companions. The mechanic gives each Doctor a "Doctor's Companion" ability that lets them partner with a companion card. This mechanically recreates the Doctor-companion relationship.

**Contrast with weaker mechanics:** Cascade on Chaos cards (Warhammer 40K) represents the unpredictability of the Warp, but it's an existing mechanic reskinned rather than a system translation. It works but doesn't feel *invented for* 40K. The strongest IP mechanics feel like they couldn't exist in any other set.

**Rule for set design:** During Phase 3 (Mechanics), ask: "What *systems* does this IP have that could become mechanics?" Not "what themes does this IP have?" Systems are interactive processes — combat systems, magic systems, relationship systems, resource systems. Themes are static concepts. Systems naturally become mechanics; themes become flavor.

Examples:
- My Little Pony's "friendship solves everything" → a mechanic where creatures get stronger when they cooperate or where friendship-based effects require controlling multiple differently-named legendary creatures
- Pokémon's evolution system → a mechanic where creatures transform into stronger versions
- Avatar: The Last Airbender's bending → a mechanic where element type (represented by color) determines what abilities a creature can unlock

## Lesson 5: Faction identity maps to color through philosophy, not aesthetics

Every successful UB set maps the IP's factions to MTG colors through their *values and methods*, and this mapping drives the entire mechanical structure:

**Warhammer 40K faction mapping:**
- Imperium (W/U/B): Order, hierarchy, discipline → Squad mechanic (organized formations)
- Tyranids (R/G): Hunger, growth, predation → Ravenous mechanic (consumption and growth)
- Necrons (U/B): Resurrection, cold logic, entropy → Unearth mechanic (self-repair)
- Chaos (R/B): Randomness, corruption, destruction → Cascade mechanic (unpredictable casting)

**LotR faction mapping:**
- Free Peoples (W/G): Community, nature, heritage → go-wide strategy
- Mordor/Sauron (B/R): Ambition, destruction, domination → Amass Orcs (vertical army building)
- Elves (U/G): Knowledge, foresight, nature → scrying mechanics
- Rohan (R/W): Passion, honor, cavalry → haste, first strike, aggression

**Rule: The faction-to-color mapping from Phase 0.5 should directly determine which mechanics each faction uses.** Don't assign mechanics randomly — a faction's mechanics should feel like the natural expression of their color philosophy as filtered through the IP's flavor.

**LotR's counter-intuitive mapping as a lesson:** Humans in LotR are the cheap mana-dork creatures (not Elves), because Tolkien's Elves are "elite yet dwindling" — there should be few of them, and they should be individually powerful. This broke from Magic's usual Elf-as-mana-dork convention because the IP demanded it. When the IP's structure conflicts with Magic's defaults, the IP wins.

## Lesson 6: Card types should match the IP's storytelling mode

**Sagas tell the IP's stories.** Every UB set uses Sagas, but the best implementations go beyond generic storytelling:

- LotR: "There and Back Again" summarizes The Hobbit in three chapters. Chapter 1 creates exactly 14 Treasure tokens (one per dwarf companion plus Bilbo). This level of obsessive flavor accuracy is what fans love.
- Doctor Who: Episode Sagas recreate specific episodes. "Blink" was the first Saga with non-linear chapter progression (I, III trigger one ability; II, IV trigger another) — because the episode is about non-linear time.
- Final Fantasy: Summon Sagas are enchantment creatures — they tell the summon's story through chapters, then stick around as a creature.

**Rule: If the IP has iconic stories/episodes/quests, use Sagas to retell them.** Each Saga should recreate a *specific* story moment, not a generic event. The chapters should mechanically mirror what happens in the story.

**Equipment represents the IP's iconic items.** LotR's Glamdring, Sting, and the One Ring. Fallout's Pip-Boy, Wasteland Survival Guide, and weapons. Final Fantasy's weapons with job-type bonuses. If the IP has iconic weapons, tools, or items, they should be Equipment cards.

**Vehicles represent the IP's transportation.** Fallout's Brotherhood Vertibird. Any IP with iconic vehicles (Star Wars X-Wings, Harry Potter broomsticks, Avatar sky bison) should use the Vehicle card type.

**Battles represent the IP's conquerable challenges.** Though no UB set has used Battles heavily yet (they were introduced in March of the Machine, a Magic-native set), they're a natural fit for any IP with boss fights, siege warfare, or overcomeable obstacles.

**Double-faced cards represent transformations.** Sephiroth transforms from Fabled SOLDIER to One-Winged Angel after 4 creatures die (a party of 4, like in the game). Any IP with character transformations, reveals, or phase changes should use DFCs.

## Lesson 7: The "knowledge pyramid" determines card distribution

The LotR design team explicitly described a **knowledge pyramid** for distributing IP references:

- **Base tier (everyone knows):** The most iconic elements — the Ring, Gandalf, Frodo, Mordor. These go at mythic and rare where they have maximum visibility and mechanical impact.
- **Mid tier (most fans know):** Major characters, important locations, key events. Faramir, Helm's Deep, the Palantir. These fill rare and uncommon slots.
- **Top tier (dedicated fans know):** Deep cuts — Bill the Pony, Bill Ferny, Butterbur the Innkeeper. These go at uncommon or appear in flavor text. They reward dedicated fans without confusing casual ones.

**Rule: Every card in the set should fall somewhere on this pyramid.** Even common cards that are "generic" creatures should use IP-specific names, creature types, and flavor text. A common 2/2 in a LotR set isn't "Gondorian Soldier" — it's "Guard of the Citadel" with flavor text from The Return of the King.

**The zero-generic-card standard:** In Tales of Middle-earth, the design team stated that *every single card* had an identifiable root in the books or movies. No card existed purely as a mechanical filler. This is the standard for IP adaptation: if a card couldn't be identified as belonging to this IP, it shouldn't be in the set.

## Lesson 8: Source-specific game mechanics deserve direct translation

When the IP is a game (video game, tabletop game), its game mechanics should be translated into MTG mechanics as directly as possible:

**Direct translations that worked:**
- Fallout's VATS → Split Second (stops opponent actions, like VATS pausing time)
- Fallout's bottle caps → Treasure tokens (1:1 conceptual mapping)
- Fallout's SPECIAL stats → seven Bobblehead artifacts (one per stat: Strength, Perception, Endurance, Charisma, Intelligence, Agility, Luck)
- Fallout's radiation → Rad counters (persistent, escalating damage)
- Fallout's junk items → Junk tokens (sacrifice to exile top card and play it)
- FF's job system → Job Select Equipment (creates Hero token, attaches equipment, grants creature types)
- FF's summons → Saga Creatures (multi-turn summoning sequence that leaves a creature behind)
- FF's leveling → Tiered mechanic (kicker variant with escalating modes)

**Rule: During Phase 3, make a two-column list: source game mechanic → MTG translation.** The left column should list every distinctive system the game has. The right column should propose how to express it in MTG's rules. Not everything will translate — but the exercise ensures you don't miss the obvious ones.

## Summary: The UB checklist

Before finishing Phase 5 for an IP-based set, verify:

- [ ] Every card has an IP-specific name (zero generic fantasy names)
- [ ] Named characters represent 20–30% of the set
- [ ] Major characters have 2–3 versions showing different story/arc moments
- [ ] Flavor text uses the right strategy (direct quotes / mixed / original in IP voice)
- [ ] At least one new mechanic is a *system translation* from the IP, not just a thematic reskin
- [ ] Faction-to-color mapping is philosophy-based and drives mechanical identity
- [ ] Sagas retell specific iconic stories/episodes from the IP
- [ ] Iconic items are Equipment cards, iconic vehicles are Vehicle cards
- [ ] Character transformations use double-faced cards
- [ ] The knowledge pyramid is respected: base-tier at mythic/rare, deep cuts at uncommon/flavor text
- [ ] Common cards reference the IP through names, creature types, and flavor text — no "generic filler"
