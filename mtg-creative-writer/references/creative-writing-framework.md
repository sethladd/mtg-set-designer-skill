# Creative Writing Framework

Operational reference for naming every card, writing flavor text, establishing naming cultures, keyword naming, and UB voice matching. This is the creative writer's handbook.

---

## The Naming Process

### Step 1: Establish the Naming Culture

Before naming any cards, define the set's linguistic identity:

1. **Choose linguistic roots** — What real-world language family anchors the names? (Germanic for horror, Greek for mythology, Japanese for feudal, Slavic for urban guilds, etc.)
2. **Define phonetic patterns** — What sounds characterize this plane? Hard consonants for aggressive worlds, liquid sounds for aquatic settings, sibilants for snake-themed cultures.
3. **Create a prefix/suffix vocabulary** — 5-10 recurring word components that mark names as belonging to this world. ("Geist-" for Innistrad spirits, "Syr" for Eldraine knights, guild names for Ravnica.)
4. **Set variety rules** — Mandate name length variety (some short, some long), construction variety (single word, compound, "X of Y"), and rhythm variety.

Document these decisions in `naming_guide.md`.

### Step 2: Name Cards by Type

**Creatures (noun phrases):**
1. Identify what the creature IS in the world
2. Choose the most specific, world-anchored descriptor
3. Apply the naming culture's linguistic patterns
4. Verify it sounds like a being, not an action

**Instants and sorceries (verb phrases/action words):**
1. Apply the [VERB] rule: "I [card name] your creature" must work
2. Choose visceral, evocative action words
3. For the set's world, use IP-specific terminology for actions

**Legendary creatures:**
1. Apply "Name, Title" format with comma
2. The title captures a SPECIFIC story moment, not a generic adjective
3. Multiple versions use different titles: "Aragorn, Company Leader" vs. "Aragorn, King of Gondor"

**Equipment:** Object name (weapon, armor, tool)
**Enchantments:** State, condition, or ongoing effect name
**Lands:** Place name, geographic feature, or named location

### Step 3: Apply the Functional Hurdles

For EVERY name, verify:
- [ ] Passes the [VERB] rule (instants/sorceries)
- [ ] Matches card type (creature names = beings, spell names = actions)
- [ ] No overlap with existing Magic card names
- [ ] No unintended Magic terminology ("tap," "ping," keyword names)
- [ ] Distinctive from other names in the same set
- [ ] Fits the naming culture (sounds like it belongs on this plane)
- [ ] ~35 characters or fewer
- [ ] Not culturally offensive or insensitively appropriated

## Flavor Text Writing

### The One-Idea Rule

Each flavor text delivers exactly ONE of these:
- An interesting idea about the world
- A haunting implication about something unseen
- A character moment that reveals personality
- A punchline that reframes the card's concept
- A piece of context that connects to the set's story

Two ideas in one flavor text = both ideas weakened. Use them on separate cards.

### The Compression Toolkit

| Tool | How to use it | When to use it |
|------|--------------|----------------|
| **Implication** | State one detail that implies a larger story | Worldbuilding — "The [thing] has been [state] since the [event]" implies the event without describing it |
| **Character voice** | Write in a character's distinctive register | Attributed quotes — personality in minimal words |
| **Twist** | Set expectation, subvert it | Humor or horror — last word changes everything |
| **Clinical detachment** | Describe terrible things neutrally | Dark humor — instructions for horrible things |
| **Scale shift** | Express cosmic insignificance | Eldrazi, cosmic horror — resignation > description |
| **Fragment** | Write as discovered artifact | Lore cards — archaeological mystery > exposition |
| **Contrast** | Juxtapose opposites | Irony — beauty/horror, power/fragility |

### Target Length

- **Sweet spot:** 8-15 words
- **Absolute maximum:** ~35 words (more is almost always too much)
- **Minimum:** 4 words can work for punchlines or declarations

### Attribution Strategy

Plan character quotes across the entire set:
1. **Identify 5-8 characters** who will have attributed quotes
2. **Assign each a voice** — distinctive speech patterns, vocabulary, attitude
3. **Distribute quotes** — each character gets 3-8 quotes across the set
4. **Build narrative** — arrange quotes so reading them in collector-number order tells a story
5. **Maintain consistency** — a character's voice should be recognizable across all their quotes

### Flavor Text Distribution

| Rarity | Target | Priority |
|--------|--------|----------|
| Mythic | 100% | Always (these are the poster cards) |
| Rare | 100% | Always (unless mechanically too complex to fit) |
| Uncommon | 40-60% | Where text box allows |
| Common | 30-50% | Where text box allows (simpler cards have more room) |

### When to Skip Flavor Text

1. Card has 8+ lines of rules text (no room)
2. Reminder text fills remaining space and is more important
3. Card is mechanically complex enough that adding text would be overwhelming

## Keyword Naming

### The Naming Spectrum Test

For every new keyword, evaluate where it falls:

| Score | Category | Test |
|-------|----------|------|
| 5 | Self-explanatory | Can a new player guess what it does from the name alone? |
| 4 | Strongly evocative | Does the name create a clear emotional/conceptual association? |
| 3 | Evocative | Does the name suggest the right general feel? |
| 2 | Opaque but memorable | Is the name memorable even if not self-explanatory? |
| 1 | Opaque and forgettable | Does the name communicate nothing? |

**Target: 4-5.** Anything below 3 needs renaming.

### Keyword Naming Principles

1. **One word if possible** — "Landfall" > "Land enters trigger"
2. **Verb-based for actions** — "Scry" (to see), "Mill" (to grind), "Investigate" (to search)
3. **Adjective-based for states** — "Indestructible," "Hexproof," "Vigilant"
4. **Resonance first** — Build on concepts players already know
5. **Test the reminder text** — If the reminder text is clear but the keyword adds confusion, the keyword name is wrong

## Universes Beyond Voice Matching

### The Voice Register Framework

Before writing any UB names or flavor text:

1. **Research the IP's dominant voice** — Read/watch source material and identify: formality level, humor ratio, vocabulary register, sentence structure, cultural references
2. **Create a voice sample** — Write 5 test flavor texts and compare against source material. If they don't sound right, adjust.
3. **Identify quote sources** — For literary IPs, compile a list of quotable passages that could serve as flavor text
4. **Establish terminology** — Make a glossary of IP-specific terms that must be used (not generic Magic equivalents)

### IP Voice Checklist

- [ ] Every card name uses IP-specific terminology (zero generic names)
- [ ] Flavor text matches the IP's formality level
- [ ] Character quotes match each character's established voice
- [ ] Humor ratio matches the IP (grimdark IP = minimal humor)
- [ ] No anachronisms (modern slang in a medieval IP)
- [ ] Source quotes used where they're stronger than original writing
- [ ] Fan-service deep cuts appear at uncommon and in flavor text

## The Complete Per-Card Creative Checklist

For every card:

1. **Name matches card type** — Creature = being, spell = action, equipment = object?
2. **Name passes [VERB] rule** (instants/sorceries)?
3. **Name fits naming culture** — Sounds like this world?
4. **Name is unique** — No existing Magic card has this name?
5. **Name length** — ~35 characters or fewer?
6. **Legendary format** — "Name, Title" with comma (if legendary)?
7. **Flavor text present** — Required at rare/mythic, encouraged elsewhere?
8. **One idea** — Flavor text delivers exactly one concept?
9. **Right type** — Narration, attributed quote, or dialogue (appropriate to card)?
10. **Attribution consistent** — Character voice matches their other quotes?
11. **Compression** — Could any words be cut without losing meaning?
12. **Tone match** — Flavor text matches card's emotional register?
13. **World-building** — Does the flavor text build the world?
14. **No contradiction** — Flavor text doesn't contradict mechanics?
